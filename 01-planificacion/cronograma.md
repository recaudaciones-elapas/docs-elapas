# Cronograma del proyecto

## Duración general

El proyecto se desarrollará en un plazo aproximado de **3 semanas**, con fecha límite de entrega del MVP funcional el **8 de mayo de 2026**.

## Distribución general del trabajo

## Semana 1
### Objetivo
Organizar el proyecto y establecer la base funcional, documental y técnica.

### Actividades principales
- Creación de la organización en GitHub
- Creación de repositorios del proyecto
- Definición del alcance del MVP
- Definición de la metodología de trabajo
- Organización del equipo
- Identificación inicial de riesgos
- Diseño preliminar de arquitectura
- Diseño preliminar del modelo de datos
- Configuración inicial del backend
- Definición de pantallas web y móvil
- Selección de tecnología para la app móvil

## Semana 2
### Objetivo
Desarrollar el flujo principal del MVP.

### Actividades principales
- Implementación de endpoints principales
- Desarrollo de interfaz web inicial
- Desarrollo de interfaz móvil inicial
- Integración entre backend, frontend web y móvil
- Registro de lecturas
- Consulta de deuda
- Registro de cortes
- Carga de datos de prueba
- Revisión de avances y ajuste de pendientes

## Semana 3
### Objetivo
Integrar, probar y preparar la entrega final.

### Actividades principales
- Corrección de errores
- Mejora básica de interfaces
- Validación del flujo completo del MVP
- Pruebas cruzadas entre móvil, backend y web
- Documentación de pruebas
- Consolidación de evidencias
- Organización de la presentación final
- Preparación de la demo

## Hitos principales

### Hito 1
Estructura inicial del proyecto y documentación base completada.

### Hito 2
Flujo principal del MVP desarrollado en backend, web y móvil.

### Hito 3
MVP funcional probado y listo para presentación.

## Observaciones

El cronograma podrá ajustarse según dificultades técnicas, siempre priorizando las funcionalidades esenciales del MVP.
