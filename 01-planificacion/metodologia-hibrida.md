# Metodología híbrida del proyecto

## Enfoque metodológico

Para el desarrollo del proyecto se adoptará una **metodología híbrida**, combinando elementos del enfoque tradicional predictivo con prácticas ágiles.

Esta decisión responde a dos necesidades:

1. Mantener una base de planificación formal, adecuada para la naturaleza académica del trabajo.
2. Permitir flexibilidad en la ejecución técnica del MVP, considerando el tiempo limitado y la evolución del desarrollo.

## Componente predictivo

El enfoque predictivo se aplicará principalmente en la fase de planificación del proyecto, incluyendo:

- definición del alcance,
- identificación de actores,
- análisis de riesgos,
- estimación de recursos,
- organización del equipo,
- cronograma general,
- documentación del proyecto.

Este componente aporta orden, claridad y control sobre lo que se realizará.

## Componente ágil

El enfoque ágil se aplicará en la fase de construcción del MVP, mediante trabajo iterativo y entregas parciales.

Se prevé aplicar prácticas como:

- división del trabajo en tareas pequeñas,
- seguimiento semanal de avances,
- revisión continua del estado del proyecto,
- priorización de funcionalidades esenciales,
- ajuste del trabajo según bloqueos o dificultades.

## Aplicación práctica en el proyecto

La metodología híbrida se aplicará de la siguiente manera:

### Fase 1: Planificación
Definición del problema, alcance, recursos, riesgos, cronograma y estructura del proyecto.

### Fase 2: Diseño técnico inicial
Definición de arquitectura general, repositorios, herramientas, modelo de datos preliminar y organización de tareas.

### Fase 3: Desarrollo iterativo
Implementación de funcionalidades esenciales del MVP mediante tareas distribuidas entre los integrantes.

### Fase 4: Seguimiento y control
Registro de avances, revisión de tareas cumplidas, identificación de retrasos y ajuste de prioridades.

### Fase 5: Pruebas y cierre
Validación del MVP, corrección de errores, organización de documentación y preparación de la presentación final.

## Ventajas de esta metodología para el proyecto

La metodología híbrida resulta adecuada porque:

- facilita la planificación formal exigida en una materia de gestión de proyectos,
- permite controlar tiempos y entregables,
- ayuda a adaptarse a imprevistos técnicos,
- favorece el trabajo colaborativo del equipo,
- se ajusta a un entorno académico con plazo corto.

## Conclusión

El uso de una metodología híbrida permite equilibrar organización y flexibilidad. De esta manera, el equipo puede planificar correctamente el proyecto y, al mismo tiempo, adaptarse durante el desarrollo del MVP sin perder el control del avance general.
