# Alcance del MVP

## Nombre del proyecto

**Sistema de Gestión de Recaudaciones y Cortes para ELAPAS - Sucre**

## Propósito del MVP

El MVP tiene como propósito demostrar la viabilidad funcional de una solución digital que conecte el trabajo de campo, el control administrativo y la consulta básica de información mediante una arquitectura compuesta por backend, web y módulo móvil.

Dado que se trata de un proyecto académico con duración limitada, el alcance se reducirá a las funciones esenciales que permitan evidenciar el flujo principal del sistema.

## Problema que se busca atender

Actualmente, los procesos de lectura, control de deuda y seguimiento de cortes pueden presentar demoras, dispersión de información o escasa trazabilidad. El sistema busca mejorar la organización de estos procesos mediante una plataforma digital centralizada.

## Funcionalidades incluidas en el MVP

El MVP incluirá las siguientes funciones principales:

### Backend
- Inicio de sesión básico
- Gestión básica de clientes
- Recepción y almacenamiento de lecturas
- Registro de cortes
- Consulta de deuda
- Endpoints para integración con web y móvil

### Web
- Inicio de sesión básico
- Consulta de clientes
- Consulta de deudas
- Visualización administrativa básica
- Revisión de registros de lecturas y cortes

### Móvil
- Inicio de sesión básico
- Búsqueda o selección de cliente
- Registro de lectura
- Registro de corte
- Adjuntar fotografía como evidencia
- Envío de datos a la API

## Funcionalidades no incluidas en esta etapa

Para mantener un alcance realista, no se incluirán en el MVP:

- Integración real con pasarelas de pago
- Facturación electrónica completa
- Geolocalización avanzada en tiempo real
- Sincronización offline compleja
- Notificaciones automáticas
- Reportería avanzada
- Seguridad empresarial completa
- Integración con sistemas institucionales externos
- Gestión avanzada de rutas de brigadistas

## Entregable esperado

Al finalizar el proyecto se espera contar con un MVP funcional compuesto por backend, web y módulo móvil básico, acompañado de documentación técnica y de gestión.

## Criterios de éxito del MVP

Se considerará exitoso el MVP si logra:

- Registrar lecturas desde el módulo móvil
- Registrar cortes con evidencia básica
- Enviar la información al backend correctamente
- Permitir consulta básica desde la interfaz web
- Tener integración funcional entre backend, web y móvil
- Contar con seguimiento documentado del proyecto
- Ser demostrable en la presentación final

## Justificación del alcance

El alcance fue definido considerando:

- el tiempo disponible hasta el 8 de mayo de 2026,
- el tamaño del equipo de trabajo,
- el carácter académico del proyecto,
- la necesidad de incluir el módulo móvil como parte obligatoria del MVP,
- la prioridad de implementar funciones críticas antes que características avanzadas.
