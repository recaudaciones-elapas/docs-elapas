# Riesgos del proyecto

## Introducción

Todo proyecto de software presenta incertidumbres que pueden afectar su desarrollo. En este caso, al tratarse de un proyecto académico con tiempo limitado, un equipo pequeño y la obligación de incluir backend, web y móvil, es importante identificar riesgos desde el inicio para tomar medidas preventivas.

## Riesgos identificados

### 1. Tiempo insuficiente para completar el alcance
**Descripción:**  
El plazo de tres semanas puede no ser suficiente para implementar todas las funcionalidades previstas en backend, web y móvil.

**Impacto:**  
Alto

**Probabilidad:**  
Alta

**Medida de mitigación:**  
Definir un alcance realista, priorizar el flujo principal del MVP y mantener el módulo móvil en una versión básica.

---

### 2. Problemas de integración entre backend, web y móvil
**Descripción:**  
Pueden surgir dificultades al conectar correctamente la API con ambas interfaces.

**Impacto:**  
Alto

**Probabilidad:**  
Alta

**Medida de mitigación:**  
Definir contratos de datos claros, endpoints mínimos y realizar pruebas de integración desde etapas tempranas.

---

### 3. Sobrecarga de trabajo en uno o más integrantes
**Descripción:**  
Al ser un equipo pequeño, la implementación simultánea de backend, web y móvil puede generar desequilibrio en la carga de trabajo.

**Impacto:**  
Medio

**Probabilidad:**  
Media

**Medida de mitigación:**  
Asignar responsabilidades claras, revisar avances de manera periódica y apoyar a quien presente bloqueos.

---

### 4. Cambios de alcance durante el desarrollo
**Descripción:**  
Pueden surgir nuevas ideas que amplíen innecesariamente el proyecto.

**Impacto:**  
Medio

**Probabilidad:**  
Alta

**Medida de mitigación:**  
Mantener el enfoque en el MVP definido y aprobar cualquier cambio antes de implementarlo.

---

### 5. Dificultades técnicas con el módulo móvil
**Descripción:**  
La configuración del entorno móvil, la captura de imágenes o la conexión con la API pueden presentar problemas.

**Impacto:**  
Alto

**Probabilidad:**  
Media

**Medida de mitigación:**  
Usar una tecnología conocida por el equipo, implementar una versión simple y evitar características móviles avanzadas.

---

### 6. Falta de seguimiento ordenado del proyecto
**Descripción:**  
Si el trabajo no se documenta ni se organiza, puede haber descoordinación y pérdida de control del avance.

**Impacto:**  
Alto

**Probabilidad:**  
Media

**Medida de mitigación:**  
Registrar tareas en GitHub, actualizar avances semanalmente y mantener la documentación al día.

## Conclusión

La gestión de riesgos permitirá anticipar problemas y tomar decisiones que favorezcan el cumplimiento del MVP dentro del tiempo establecido. El principal criterio será mantener un alcance realista y asegurar una buena coordinación entre backend, web y móvil.
